Features
Everything you need. Nothing you don't.

Built from the best ideas in the Claude Code ecosystem, distilled into one shell script.

üß†

Self-Authored Handoffs

Each session writes NEXT_PROMPT.md ‚Äî a bespoke briefing with exact resume points, tips, warnings, and setup steps. Not a template. A custom briefing that gets better every time.

üìö

Knowledge Accumulation

KNOWLEDGE.md has two sections: Core Knowledge (permanent ‚Äî never pruned) and Session Log (auto-pruned to the last 5 entries). Claude promotes durable insights to Core; stale session notes are automatically dropped. Session 30 gets session 1's best discoveries without wading through 29 sessions of noise.

Inspired by claude-session-init
‚úÖ

Objective Verification

With --verify, provide a command that objectively checks completion. Tests pass? Script exits 0? Task is done ‚Äî regardless of what the agent claims. Don't trust self-assessment.

Inspired by Ralph Wiggum's completion promise
‚è≥

Rate Limit Resilience

Detects rate limits from Claude Code output, displays a countdown timer, waits for the reset, then retries the same session. No session wasted on a rate limit exit.

Inspired by claude-auto-resume
üéØ

Dual Context Threshold

Research shows quality degrades above 50% context usage. Cleave uses a two-phase approach: productive work stops at 60%, handoff writing happens in the 60‚Äì70% buffer zone (structured output tolerates mild decline), and 70%+ is a hard wall. ~20% more productive capacity per session than a single-threshold approach.

Informed by GSD framework research
üîÄ

Subagent Spawning

With --subagents, hints Claude Code to spawn claude -p headless subagents for heavy subtasks. Main session stays lean at ~15% while subagents get fresh 200K windows.

Informed by GSD's lean orchestrator pattern
üîÑ

Loop Detection

Compares consecutive handoffs. If >85% identical lines, warns you. After 3 loops, stops automatically. No more burning tokens on a stuck agent.

üîî

Desktop Notifications

macOS and Linux alerts for task completion, rate limit waits, loop detection, and max sessions. Know what's happening without watching the terminal.

üì¶

Git Checkpoints

With --git-commit, auto-commits after each session. Roll back to any point with git revert. Every session is a safe checkpoint.

üìä

Full Audit Trail

Every prompt, progress report, handoff, and knowledge snapshot archived in .cleave/logs/. Debug any session. Replay any step.

üõ°Ô∏è

Crash Recovery

If Claude Code crashes without writing a handoff, Cleave falls back to your initial prompt + PROGRESS.md + KNOWLEDGE.md. No work is lost.

‚ö°

Zero Dependencies

One shell script. No npm, no pip, no framework, no API keys, no config. If you have Bash and Claude Code CLI, you're ready.

Risks & Mitigations
Autonomous agents need guardrails.

Honest about the risks. Clear about the mitigations.

‚ö†Ô∏è Cost Awareness

Each session costs API tokens. A 15-session Opus run: $75‚Äì225. Always set --max-sessions. Start with 3‚Äì5 to test. Check status.json for live progress.

üõë Permission Bypass

Runs with --dangerously-skip-permissions. Claude can execute any command. Use --git-commit for rollback safety. Run in feature branches, never on main.

‚ö†Ô∏è Handoff Quality

The agent usually writes excellent handoffs ‚Äî but sometimes vague ones. Crash recovery (initial prompt + PROGRESS.md + KNOWLEDGE.md) catches failures. Check first 2‚Äì3 sessions.

‚ÑπÔ∏è Prompt Quality Matters

Cleave chains sessions ‚Äî it doesn't plan your task. A bad initial prompt produces bad sessions. Invest time in a specific, well-structured prompt.

üõë Rate Limits

Auto-waits up to 5 hours. On free tiers, long runs may stall repeatedly. Use --pause 60 for longer gaps. Cleave always retries ‚Äî never wastes a session.

‚ÑπÔ∏è Loop Detection

If the agent gets stuck, consecutive handoffs will be near-identical. Cleave auto-stops after 3 loops. Check .cleave/logs/ and modify your prompt.

Quickstart
Three steps to infinite context.
1

Write your task prompt

Describe the task, files, tools, and what "done" looks like.

cat > my_task.md << 'EOF'
Convert 500 test files from unittest to pytest.
Tests are in ./tests/. Start alphabetically.
For each file: convert, run, verify it passes.
EOF
2

Install Cleave

One file. Download, chmod, done.

curl -O https://raw.githubusercontent.com/REPO/cleave/main/cleave
chmod +x cleave
sudo mv cleave /usr/local/bin/
3

Launch and walk away

Sessions chain automatically. Come back to finished work.

cleave --max-sessions 20 --verify "pytest -x" --git-commit my_task.md
Standing on Shoulders
Credit where it's due.

Cleave is a synthesis. These projects contributed the ideas that make it work.

Ralph Wiggum

‚Üí Verification-first philosophy

The insight that objective verification (tests pass, build succeeds) beats agent self-assessment. "Completion promise" pattern for clean exit signals.

GSD (Get Shit Done)

‚Üí Context rot research, subagents

Quantified context degradation: 0‚Äì30% = peak, 50%+ = declining, 70%+ = hallucinations. Dual threshold model and lean orchestrator pattern with fresh subagent contexts.

claude-auto-resume

‚Üí Rate limit detection

Parsing Claude output for rate limit messages, calculating wait time from reset timestamps, countdown display. The pattern we adapted for Cleave's main loop.

claude-session-init

‚Üí Persistent knowledge files

The concept of structured knowledge files that accumulate across sessions ‚Äî with auto-compaction to prevent context bloat.
